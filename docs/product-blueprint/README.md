# CULO Product Blueprint

The Product Blueprint is the implementation layer of CULO. Where the Constitution defines what CULO stands for, the Blueprint defines how CULO is built.

Every developer, designer, AI assistant and future team member should read this document set before building features for CULO.

> **Rule:** From this point forward, every time a feature is built, the prompt should begin with:
> *"Follow the CULO Constitution, Product Blueprint and Business Rules. If a requested feature conflicts with those documents, report the conflict before making changes."*

## How to Use the Blueprint

**Building a new feature?** Start with Part 01 for context, then read the chapter most relevant to your feature.

**Defining permissions?** See Part 02 — User Types.

**Building a publishing flow?** See Part 03 — Publisher Workflow.

**Designing the data model?** See Part 04 — Knowledge Graph and Part 13 — API & Data Model.

**Adding a monetisation feature?** See Part 10 — Monetisation and Part 12 — Business Rules.

**Setting up automations?** See Part 11 — Automations and Part 03 — Publisher Workflow.
